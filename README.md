# HBM Nuclear Server Pack

Репозиторий для сборки Minecraft-сервера вокруг Hbm's Nuclear Tech Mod с разделением на клиентскую и серверную части.

## Целевая платформа

- Minecraft: `1.12.2`
- Mod loader / server core: `Forge 14.23.5.2860`
- Основной тестовый мод: `HBM's Nuclear Tech Mod: Community Edition`
- Серверная ОС: `Ubuntu Server 22.04 LTS`, без GUI
- Плагинный слой: `SpongeForge` как отдельный серверный профиль, если реально нужны Sponge-плагины

Почему не `1.7.10`: оригинальная ветка HBM сильная, но серверное ядро с плагинами для 1.7.10 обычно завязано на старые и плохо поддерживаемые Cauldron/KCauldron/Thermos-подобные решения. Для своего сервера безопаснее стартовать с Forge-сборки и подключать плагины через Sponge там, где это возможно.

## Структура

```text
docs/
  BOOT_PROFILE.md           # минимальный первый HBM CE boot-test
  DECISIONS.md              # выбранные версии и технические решения
  FIRST_DEPLOY_CHECKLIST.md # первый headless деплой на Ubuntu
  MODLIST.md                # человекочитаемый список модов
  PLUGINLIST.md             # список серверных плагинов
  SERVER_CLIENT_SPLIT.md    # правила разделения клиента и сервера
  UBUNTU22_SERVER.md        # подготовка headless Ubuntu 22.04 сервера
manifests/
  mods.csv                  # машинно-читаемый реестр модов
  plugins.csv               # машинно-читаемый реестр плагинов
profiles/
  client/
    mods/                   # клиентские .jar, не коммитить
    config/                 # клиентские конфиги
    resourcepacks/          # ресурспаки, не коммитить крупные архивы
  server/
    mods/                   # серверные .jar, не коммитить
    config/                 # серверные конфиги
    plugins/                # Sponge-плагины, не коммитить
    world/                  # локальный мир, не коммитить
    *.example               # шаблоны server.properties, eula, start.sh, systemd
  common/
    config/                 # конфиги, которые должны совпадать у клиента и сервера
scripts/
  README.md                 # место для скриптов установки/проверки
```

## Правило репозитория

В репозитории храним манифесты, конфиги, документацию и скрипты. Файлы `.jar`, миры, логи и кэши не коммитим. Когда список модов будет подтвержден тестовым запуском, можно добавить скрипт, который собирает клиентский и серверный профили из `manifests/`.
