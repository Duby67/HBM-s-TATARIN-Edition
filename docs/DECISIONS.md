# Technical Decisions

## Базовая версия

Выбрана платформа:

- Minecraft `1.12.2`
- Forge `14.23.5.2860`
- Java `8`
- OS target: `Ubuntu Server 22.04 LTS`, headless

Причины:

- HBM NTM: Community Edition выглядит наиболее актуальной тестовой HBM-линией для `1.12.2`.
- Для `1.12.2` есть зрелая экосистема технических, QoL и серверных модов.
- Forge-сервер проще поддерживать, бэкапить и переносить, чем гибридные Bukkit/Forge-ядра старых версий.

## HBM Boot Line

Первая тестовая HBM-линия зафиксирована:

- `HBM's Nuclear Tech Mod: Community Edition`
- Без CE Space
- Без Leafia addon
- Без SpongeForge
- Без оптимизаторов
- Без клиентских QoL-модов

Подробный boot-profile: `docs/BOOT_PROFILE.md`.

## Серверное ядро

Основной профиль:

- `Forge 14.23.5.2860`
- Только Forge-моды
- Без Bukkit/Spigot/Paper-плагинов

Профиль с плагинами:

- `Forge 1.12.2` + `SpongeForge`
- Плагины только Sponge API 7
- Перед продакшеном обязательно прогнать совместимость HBM + SpongeForge + серверные плагины

Нюанс по SpongeForge: stable-канал Sponge для `1.12.2` указывает сборку `1.12.2-2838-7.4.7`, а CurseForge LTS показывает более новый `spongeforge-7.5.0.jar`. Поэтому плагинный профиль надо тестировать отдельно:

1. Сначала чистый Forge `14.23.5.2860` без Sponge.
2. Затем профиль SpongeForge LTS.
3. Если будут mixin/coremod-конфликты, откатить плагинный профиль к Forge build `2838`, не меняя основной Forge-профиль сборки.

Важно: Bukkit/Spigot/Paper-плагины нельзя просто положить в Forge-сервер. Для плагинов нужен SpongeForge или отдельное гибридное ядро. Гибридные ядра не берем как базовую линию, потому что они повышают риск крашей, дюпов и проблем с модовыми блоками.

## Разделение сборок

Клиент получает:

- все общие моды из `common`
- клиентские QoL-моды
- визуальные и интерфейсные моды
- ресурспаки

Сервер получает:

- все общие сервер-совместимые моды
- серверные моды управления, профилирования и защиты
- SpongeForge и Sponge-плагины только в отдельном профиле

## Статусы манифеста

- `required` - нужен для запуска сборки
- `recommended` - почти всегда стоит включать
- `optional` - подключать после базового теста
- `candidate` - требует проверки совместимости
- `rejected` - не использовать, причина в заметках
